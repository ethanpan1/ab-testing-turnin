A/B Testing

For this assignment, I implemented an A/B test for a mock cactus shop website. I practiced data collection for A/B tests, as well as skills in statistical analysis, specifically two-sample t-tests and chi-square tests.

Part 0: Creating Hypotheses for Statistical Tests

The first part of the assignment was to modify a template to create two versions of a cactus shop website. These two versions would be used for our A/B test. I've included screenshots of both pages, as well as the link to the deployed app.

In particular, I was curious whether really minute changes in the interface would have significant effects on the metrics I'd be testing. As such, I changed the subheading of the page, the font of the price (italic vs. bold), and the color of the item count. While I changed the images to be more aesthetically pleasing, they and their layout were the same in both interfaces.

Part 1: Creating Hypotheses for Statistical Tests

I ended up testing two metrics for my A/B test. The first was the time to complete the task: adding at least $150 of cacti to the cart. The second was the return rate: how often users went to the checkout page and back to the buying page. I created null and alternative hypotheses for both.

Time to Complete

Null hypothesis: Users do not differ in the amount of time spent buying cacti between interface A and interface B.

Alternate hypothesis: Users spend less time buying cacti in interface B than in interface A.

Explanation: Prices are bolded rather than italicized, making them more noticeable or easier to find. Subheading is shortened and less interesting, so users may spend less time reading it.

Return Rate

Null hypothesis: Users do not differ in how frequently they return to the cactus buying page between interface A and interface B.

Alternate hypothesis: Users more often return to the cactus buying page in interface A than in interface B.

Explanation: The “X in cart” counter is green in interface A, highlighting it and potentially encouraging users to return and look at the item counts. This likely would not change how often users underspend in their first attempt at the task, but could change whether they choose to go back anyway.

Part 2: Collecting Data and Applying Scripts

As described above, I tasked users to add at least $150 worth of cacti to the cart. I fielded users through Piazza as well as by messaging my friends. I then extracted user logs through Heroku and filtered them. I proceeded to preprocess the data and run it through scripts that applied the statistical tests.

Part 3: Conclusions and Takeaways

I created an infographic to display the results of the tests. As it states, interfaces A and B were not statistically different in time to completion or return rate.
Some takeaways from the assignment:
  • Even if the means may suggest significance, it all comes down to sample size. My sample size was very low, so there wasn’t enough statistical power to substantiate the difference in means.
  • My users were not acting entirely naturally, especially since my classmates participated in my A/B test among many others. This limits the results.
  • It is still unclear whether such minute changes can have significant effects on user behavior. Larger design changes, like layout or navigation, are more likely to have significant effects.
